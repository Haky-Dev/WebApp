# PC 반응형 UI 개선 설계

**날짜**: 2026-06-05  
**대상 페이지**: 배정 결과 페이지, 배정 애니메이션, 그룹 추첨 세리머니

---

## 1. 목표

- PC(768px 이상)에서 레이아웃을 넓게 확장하고 폰트를 키워 가독성 향상
- 팀 카드에 정보 밀도 극대화 (동호회, 이름, 레이팅, 합산 레이팅 모두 표시)
- 배정 결과 페이지에 통계 헤더 + 정렬/필터 추가
- 스핀 애니메이션을 슬롯머신 + 블러 + 바운스+글로우 조합으로 강화
- 모바일은 현재 UI 그대로 유지

---

## 2. 공통 기반

### `hooks/useIsDesktop.ts`

```ts
// 768px 이상이면 true를 반환하는 훅
export function useIsDesktop(): boolean
```

- `useState` + `useEffect`로 `window.matchMedia('(min-width: 768px)')` 구독
- SSR-safe: 초기값 `false`, 클라이언트 마운트 후 실제 값으로 업데이트
- 세 페이지 모두 이 훅 하나로 PC 여부 판단

### 브레이크포인트

| 환경 | 기준 |
|------|------|
| 모바일 | `< 768px` |
| PC | `≥ 768px` |

## 3. 배정 결과 페이지 (`/events/[id]/results`)

### 3-1. 컨테이너

| | 모바일 | PC |
|-|--------|----|
| maxWidth | `512px` | `1100px` |
| padding | `24px` | `40px 48px` |

### 3-2. 통계 헤더 (`AllResultsTab` 상단, PC 전용)

3개 수치를 가로 배치:

```
┌─────────────────────────────────────────────────────┐
│  총 12팀          24명 참가          평균 레이팅 8.45  │
└─────────────────────────────────────────────────────┘
```

- 수치: `fontSize: 24px`, `fontWeight: 900`, `color: var(--text-primary)`
- 라벨: `fontSize: 11px`, `fontWeight: 700`, `color: var(--text-muted)`
- 배경: `var(--bg-surface)`, `borderRadius: 10px`, `padding: 16px 20px`
- 계산 방법:
  - 총 팀 수: `pairs.length`
  - 참가자 수: `pairs.length * 2`
  - 평균 레이팅: 전체 참가자 레이팅 평균 (소수점 2자리)

### 3-3. 정렬/필터 바

검색창 아래에 한 줄로 배치:

- **정렬** (select):
  - `팀 번호순` (기본)
  - `합산 레이팅 높은순`
  - `합산 레이팅 낮은순`
- **동호회 필터** (select):
  - `전체` (기본)
  - pairs에서 자동 추출한 동호회 목록 (알파벳/가나다 정렬)
  - 동호회가 없는 참가자는 무시

모바일에서도 표시 (검색창 아래 줄바꿈 없이 가로 배치).

### 3-4. 팀 카드 그리드

| | 모바일 | PC |
|-|--------|----|
| 레이아웃 | flex column | `grid-template-columns: repeat(2, 1fr)` |
| 카드 gap | `7px` | `12px` |

### 3-5. 팀 카드 내용

참가자 정보 표시 형식 (양쪽 모두):
```
동호회명          ← fontSize: 모바일 10px / PC 12px, color: var(--text-muted)
이름  8.25        ← 이름 fontSize: 모바일 15px / PC 20px / fontWeight: 800
                    레이팅은 이름 우측, fontSize: 모바일 11px / PC 14px
```

카드 우측 상단에 합산 레이팅 배지:
```
합산 16.50        ← fontSize: 11px, fontWeight: 800, color: var(--text-muted)
```

전체 카드 구조:
```
┌── A1 ──────────────────────── 합산 16.50 ─┐
│  행복클럽        ×       서울클럽           │
│  김철수  8.25            이영희  8.25       │
└────────────────────────────────────────────┘
```

- `×` 구분자는 flex 중앙 정렬 유지
- 동호회가 없으면 해당 라인 생략

## 4. 배정 애니메이션 (`AssignmentAnimation`)

### 4-1. 드럼롤 단계

- 컨테이너 maxWidth: `480px` → PC에서 `700px`
- 이름 폰트: `30px` 고정 → `clamp(30px, 4vw, 64px)`
- **공개 후 하단 목록 형식 변경**:
  - 현재: `김철수 × 이영희`
  - 변경: `행복클럽 김철수 (8.25) × 서울클럽 이영희 (8.25)`
  - 동호회 없으면 `김철수 (8.25) × 이영희 (8.25)`

### 4-2. 스핀 애니메이션 (드럼롤 단계)

슬롯머신 + 블러 + 바운스+글로우 조합:

**스핀 중**:
- 이름 배열을 빠른 속도로 위→아래 방향으로 순환 표시 (slot machine)
- `filter: blur(2px)` 적용
- `opacity: 0.7`

**감속 구간** (속도 느려질 때):
- blur 점차 해제 (`blur(2px)` → `blur(0)`)
- opacity 1로 복원

**확정 순간**:
- `transform: scale(1) → scale(1.15) → scale(1)` 바운스 (duration: 300ms)
- 네온 글로우 펄스 1회: `text-shadow: 0 0 20px currentColor`

세리머니의 추첨 단계에도 동일한 애니메이션 적용.

### 4-3. 피날레 단계

| | 모바일 | PC |
|-|--------|----|
| 레이아웃 | 1열 | 2열 그리드 |
| maxWidth | `480px` | `900px` |
| 카드 pair text | `16px` | `20px` |

카드 구조 (피날레):
```
┌─── T1 ────────── 합산 16.50 ─┐
│  행복클럽    서울클럽          │
│  김철수 8.25 × 이영희 8.25    │
└──────────────────────────────┘
```

## 5. 그룹 추첨 세리머니 (`GroupDrawCeremony`)

### 5-1. 그리드 단계 (그룹 선택)

| | 모바일 | PC |
|-|--------|----|
| 최대 열 수 | `Math.min(groups.length, 4)` | `Math.min(groups.length, 6)` |
| 버튼 minWidth | `120px` | `160px` |
| 그룹 글자 | `clamp(36px, 5vw, 64px)` | `clamp(36px, 5vw, 80px)` |

### 5-2. 추첨 단계 (스핀 공개)

- 스핀 애니메이션: 섹션 4-2와 동일 (슬롯머신 + 블러 + 바운스+글로우)
- 팀A 이름 아래에 동호회·레이팅 추가:
  ```
  행복클럽
  김철수  8.25     ← 이름 clamp(30px, 6.5vw, 86px), 레이팅 14px
  ```
- 스핀 중인 상대방도 동일 형식

### 5-3. 피날레 단계

| | 모바일 | PC |
|-|--------|----|
| 열 수 | `3열` | `5열` |
| maxWidth | `900px` | `1200px` |

카드 구조:
```
┌─── A1 ── 합산 16.50 ─┐
│  행복클럽             │
│  김철수  8.25         │
│  서울클럽             │
│  이영희  8.25         │
└──────────────────────┘
```

- 동호회 없으면 해당 줄 생략
