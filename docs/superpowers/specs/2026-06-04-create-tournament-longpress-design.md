# 토너먼트 개설 롱프레스 진입점 설계

**날짜:** 2026-06-04  
**범위:** `app/page.tsx` 수정

---

## 개요

홈 화면 "진행 중인 토너먼트" 섹션 타이틀을 롱프레스(3초)하면 토너먼트 개설 모달이 열린다. 기존 마스터 어드민 롱프레스(로고 3초)와 동일한 인터랙션 패턴을 재사용한다. 마스터 인증 없이 토너먼트 이름과 관리자 PIN만 입력하면 바로 개설할 수 있다.

---

## 트리거

- **대상 요소:** "진행 중인 토너먼트" 섹션 타이틀 텍스트
- **인터랙션:** 3초 롱프레스 (마우스/터치 모두 지원)
- **조건:** 마스터 어드민 로그인 여부와 무관하게 항상 동작

---

## 모달 UI

기존 마스터 PIN 모달(`modal-overlay` / `modal-panel`)과 동일한 스타일 사용.

| 요소 | 내용 |
|------|------|
| 상단 레이블 | `🏆 토너먼트 개설` (accent 색상, uppercase) |
| 제목 | `새 토너먼트 만들기` |
| 필드 1 | 토너먼트 이름 (text, placeholder: "예: 봄 클럽 드래프트", required) |
| 필드 2 | 관리자 PIN (password, placeholder: "••••", required, minLength: 4) |
| 에러 메시지 | API 오류 시 accent-danger 색상으로 표시 |
| 버튼 | `+ 만들기` (btn-cta) / `취소` (btn-ghost) |

---

## 데이터 흐름

1. 롱프레스 감지 → `showCreateModal` state를 `true`로 설정
2. 폼 제출 → 기존 `handleCreate` 로직과 동일하게 `POST /api/events` 호출
3. 성공 → `router.push(/events/${event.id})`로 이동
4. 실패 → 모달 내 에러 메시지 표시, 모달 유지
5. 취소 → 모달 닫기, 입력값 초기화

---

## 상태 관리

`page.tsx`에 추가할 state:

```ts
const [showCreateModal, setShowCreateModal] = useState(false)
```

기존 `name`, `pin`, `creating`, `createError` state는 재사용.  
모달 닫을 때 `name`, `pin`, `createError` 초기화.

롱프레스 타이머는 기존 `pressTimer` ref와 **별도** ref(`createPressTimer`)로 관리해 충돌 방지.

## 기존 기능과의 관계

| 기능 | 진입점 | 변경 여부 |
|------|--------|-----------|
| 마스터 어드민 로그인 | 로고 3초 롱프레스 | 변경 없음 |
| 마스터 어드민 패널 (동호회 관리 + 토너먼트 생성) | 마스터 로그인 후 자동 표시 | 변경 없음 |
| **토너먼트 개설 모달** | "진행 중인 토너먼트" 3초 롱프레스 | **신규** |

---

## 고려사항

- 마스터 어드민 상태(`isMasterAdmin`)일 때도 롱프레스 동작 — 중복 기능이지만 일관된 UX
- 모달이 이미 열려 있으면 롱프레스 타이머 시작하지 않음
- 롱프레스 중 스크롤 등으로 터치가 취소되면 타이머 해제
